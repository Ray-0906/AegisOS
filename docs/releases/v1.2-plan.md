# AegisOS v1.2 Engineering Roadmap

AegisOS is transitioning from a feature-driven prototype into a maintained distributed systems platform. v1.2 consolidates stabilization, CI hardening, and engineering infrastructure into a single release.

## Milestones
- **v1.1** → Frozen release ✅
- **v1.2** → Stabilization & Engineering Infrastructure
- **v1.3** → Runtime Persistence
- **v1.4** → Security & Trusted Images
- **v1.5** → Persistent Containers (CRIU)

---

## v1.2 Sprint Plan

### Sprint 1: Deterministic Tests (Highest Priority)
Fix the P0 flaky release blockers and introduce synchronization primitives.

**Primitives to introduce:**
- `TestBarrier` — node startup synchronization
- `EventAwaiter` — state/replication event synchronization
- `TestClock` — controllable time for lease-based tests
- `ClusterAwaiter` — centralized cluster wait patterns

**Immediate targets:**
- Fix `CorruptCheckpointRecoveryTest` (P0)
- Fix `Phase7Test` flakiness (P0)
- Create `TEST_DEBT.md` classifying all production test sleeps

### Sprint 2: CI Profiles & System Invariants
Split the test suite into purpose-driven Maven profiles.

| Profile       | Contents                              | Target Time | Trigger  |
| ------------- | ------------------------------------- | ----------- | -------- |
| `fast`        | Unit + deterministic integration      | < 5 min     | Every PR |
| `integration` | All stable integration tests          | 10-20 min   | Nightly  |
| `chaos`       | Phase7/8, WorkerFailure*, LongRunning | 20-40 min   | Weekly   |
| `quarantine`  | Known flaky tests under active fix    | Variable    | Weekly   |

Finalize `SYSTEM_INVARIANTS.md` and `ADR-023`.

### Sprint 3: Runtime Persistence Foundation
Build architectural primitives only (no actual persistence):
- `ExecutionJournal`
- `LeaseJournal`
- `OwnershipJournal`

### Sprint 4: Remaining Sleep Replacement
Replace P1/P2 `Thread.sleep` usages with the new primitives.

---

## Architecture Boundaries (Frozen)
- `ProcessRuntimeAgent` must not know about container IDs.
- `DockerRuntimeBackend` must not know about Raft.
- `JvmRuntimeBackend` must not know about scheduling.
- `MetricsServer` is read-only forever.
