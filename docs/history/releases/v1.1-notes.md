# AegisOS v1.1 Release Notes

## Overview
AegisOS v1.1 marks a major stabilization milestone, bringing significant architectural hardening and introducing container-based execution capabilities alongside the existing JVM workload model.

## Major Changes
- **Dual Runtime Architecture**: Successfully decoupled the execution environment from the orchestration layer. AegisOS now supports both JVM-native and Docker container executions.
- **Worker Lifecycle Hardening**: Increased resilience for distributed worker failures and race conditions, ensuring robust distributed job state tracking.
- **Log Compaction**: Implemented Raft Log Compaction for bounded consensus storage overhead.
- **Observability**: Added a mature, in-memory bounded `/jobs` and `/jobs/{id}` topology API, exposing the timelines of workloads without polluting Raft consensus with metric cardinality.

## Architectural Decisions
- **[ADR-020] RPC Correlation Isolation**: Strict RPC request-response boundaries were created to guarantee future completions.
- **[ADR-021] Container Semantics**: Containers run under deterministic configurations and environment variables instead of loosely coupled map-based properties.
- **[ADR-022] Runtime Ownership**: Enforced a strict boundary where `ProcessRuntimeAgent` orchestrates, but specific `RuntimeBackends` perform the physical execution.

## Known Limitations
- **CorruptCheckpointRecoveryTest**: This chaos test exhibits determinism issues due to tight distributed timings between node failure and chunk replication. Excluded via quarantine profile until v1.2.
- **Container Semantics**: Container jobs are currently restartable, not resumable. Upon failure, they start from scratch.
- **Trusted Images**: The Trusted Image policy is rudimentary and relies on string-based domain validation rather than OCI image signatures or digests.

## Regression Coverage
The AegisOS v1.1 release gate successfully passed 98 integration/chaos tests. 1 test has been quarantined.

## Upgrade Path
Drop-in replacement for v1.0. All previous `AegisJob` interfaces are fully backwards compatible, mapped implicitly to `RuntimeType.RUNTIME_JVM`.

## Future Roadmap
- **v1.2**: Persistent runtimes, fixing technical debt like `CorruptCheckpointRecoveryTest` determinism.
- **v1.3**: Security enhancements, including signed OCI images, quotas, and CRIU support.
- **v1.4**: Multi-tenancy isolation.
